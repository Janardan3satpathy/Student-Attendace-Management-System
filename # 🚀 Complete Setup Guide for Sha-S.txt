## ðŸ”§ Advanced Configuration

### Custom Domain (Streamlit Cloud Pro)
1. Go to your app settings
2. Click "Custom domain"
3. Add your domain name
4. Update DNS records as instructed

### Environment Variables
If you need to add secrets (API keys, etc.):

1. In Streamlit Cloud, go to app settings
2. Click "Secrets"
3. Add your secrets in TOML format:
```toml
[database]
connection_string = "your_connection_string"

[api]
key = "your_api_key"
```

4. Access in your code:
```python
import streamlit as st
db_connection = st.secrets["database"]["connection_string"]
```

### Custom Requirements
Add specific package versions in `requirements.txt`:
```text
streamlit==1.31.0
sqlalchemy==2.0.25
pandas==2.2.0
werkzeug==3.0.1
pillow==10.2.0  # Add if you need image processing
```

## ðŸ› Troubleshooting

### Issue: "ModuleNotFoundError"
**Solution:**
```bash
pip install -r requirements.txt --upgrade
```

### Issue: Database Not Created
**Solution:**
- Ensure `attendify.db` is in `.gitignore`
- The database will be created automatically on first run
- Check file permissions in the directory

### Issue: Streamlit Cloud Deployment Fails
**Solution:**
1. Check deployment logs
2. Verify `requirements.txt` syntax
3. Ensure all imports are available in pip
4. Check Python version compatibility

### Issue: App Runs Locally but Not on Cloud
**Solution:**
1. Check for hardcoded file paths (use relative paths)
2. Verify no local dependencies
3. Check logs on Streamlit Cloud
4. Ensure no `.env` or secret files are required

### Issue: Database Not Persisting on Streamlit Cloud
**Solution:**
- SQLite databases don't persist on Streamlit Cloud
- Use Streamlit's secrets for connection strings
- Consider upgrading to PostgreSQL:
```bash
pip install psycopg2-binary
```

Update database URL:
```python
# In app.py, change:
engine = create_engine('sqlite:///attendify.db', echo=False)

# To:
import streamlit as st
engine = create_engine(st.secrets["database"]["url"], echo=False)
```

### Issue: Slow Performance
**Solution:**
1. Use `@st.cache_data` for expensive computations
2. Optimize database queries
3. Add indexes to frequently queried columns
4. Consider pagination for large datasets

## ðŸ” Security Best Practices

### 1. Change Default Passwords
Before deployment, update seed data in `app.py`:
```python
admin.set_password('your_secure_admin_password')
teacher.set_password('your_secure_teacher_password')
student.set_password('your_secure_student_password')
```

### 2. Use Environment Variables
For sensitive data:
```python
import os
SECRET_KEY = os.getenv('SECRET_KEY', 'default_dev_key')
```

### 3. Enable HTTPS
Streamlit Cloud provides HTTPS by default.

### 4. Implement Rate Limiting
Add rate limiting for login attempts:
```python
if 'login_attempts' not in st.session_state:
    st.session_state.login_attempts = 0

if st.session_state.login_attempts > 5:
    st.error("Too many login attempts. Try again later.")
    return
```
